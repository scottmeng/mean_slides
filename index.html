What does "MEAN" mean?

  • M: MongoDB - noSQL document-based data base
  • E: Express - Node.JS web application framework
  • A: AngularJS - client-side MVC framework
  • N: Node.JS - server JS based on Google's V8 engine

MongoDB

  • Document Oriented, data stored in JSON like documents
  • Schemaless
  • Good integration with Node.JS through mongoose

Express

  • Light-weighted and robust framework
  • Rendering with template engine
  • Routing

AngularJS

  • Client side framework following MVC pattern
  • Client side routing and templating
  • Dependency injection
  • Data binding (two-way!)
  • Developed by Google

Node.JS

  • Javascript for backend
  • Event-driven
  • Non-blocking IO (callbacks everywhere :o)

Why so MEAN?

  • Never changing language
  • Never changing objects
  • Fast development and iteration
